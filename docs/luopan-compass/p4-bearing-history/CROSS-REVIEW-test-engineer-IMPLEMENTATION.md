# Cross-Review: Test Engineer — Implementation
Feature: p4-bearing-history
Date: 2026-04-28
Reviewer Role: Test Engineer
Document Reviewed: Implementation (codebase + test suite)
Reference: REQ-luopan-p4-bearing-history.md, PROPERTIES-luopan-p4-bearing-history.md

## Summary

The unit and integration test suite for the drift detection subsystem, DAO contract, ViewModel logic, sensor logging, and database migration is thorough and well-structured. A significant cluster of E2E properties covering the age-based recalibration banner, banner dismiss/re-show lifecycle, CalibrationWizard RESULT_OK paths, and swipe-to-delete edge cases are blocked by the absence of Hilt DI and are currently `@Ignore`-annotated with accurate explanations — these represent real coverage gaps that cannot be deferred beyond Phase 5 without product risk.

## Recommendation

**Need Attention**

Three High findings must be resolved before Phase 4 is declared done:

- **F-01**: Clarify the PROP-HIST-064 reclassification — either update PROPERTIES to reflect the unit-level `computeListState()` approach (preferred, since states A and B are already covered by AT-HIST-04-A and AT-HIST-02-E), or add the missing Espresso sub-tests.
- **F-02**: Add the PROP-HIST-042 Snackbar 5-second dismissal timing test — this is a user-observable contract that the implementation satisfies but no test verifies.
- **F-03**: Unblock AT-HIST-01-C (mutual exclusion) and AT-HIST-05-C (expanded deviation format) without Hilt — both are achievable with current test infrastructure by seeding the real encrypted DB from `@Before`. The remaining 20+ `@Ignore`d tests are accepted as Phase 5 Hilt-migration debt and must be tracked in a ticket.
